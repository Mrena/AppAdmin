Here you can choose whether you want to be able to change the settings later, or want to set it once and then have the settings button, and page disappear.

The One Time option is useful when you don't want to have your client play around with the settings, and is good for security reasons.

The Changable option is good for when you are managing your own or client's web app/website and you might change the page which will receive query strings, or the query strings themselves.

Settings

  • URL to query:
  • This is the URL where your server-side script is hosted.
  • Page e.g url.com:
  • This is the page that will handle query string send for data such as messages. It should check to see if a query is valid, and then redirect the data to a method which will retrieve and return the required data.
  • URL parameter name:
  • This is the page that will handle query string send for data such as messages. It should check to see if a query is valid, and then redirect the data to a method which will retrieve and return the required data.
  • Text Fields
  • Email Address value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded email address you want editable.
  • Blog Address value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded blog address you want editable.
  • Home Summary value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded home summary string you want editable.
  • Home Page description value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded home page description you want editable.
  • About Us description value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded about us description you want editable.
  • Building No. value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded building number of the company or person's address you want editable.
  • City value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded city of the company or person's you want editable.
  • Country value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded country of the company or person's you want editable.
  • Logo value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded path of the logo you want editable.
  • Messages value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded array of objects with fields 'Message_m' which should contain a message, 'message_id' which should the message's id and 'From_user' which should contain the email address of the person the message is from you want editable.
  • Audio value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded array which contains paths to the audio you want editable.
  • Photos value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded array which contains paths to the photos you want editable.
  • Videos value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded array which contains paths to the videos you want editable.
  • Stats value:
  • When your script receives a query string with a value you assigned in this field, you should return a JSON encoded array who's first index contains an array which contains the number of visits to the app or website in the first index of the array, and the number of unique visits as the second index you want editable. The second index should contain an object which has a name of the country as field, and the statistics as a value.

Actions

  • Editing Text:
  • When a user edits any of the text fields in the text management section, and tap save. Your server-side script will recieve a query string who's name is the one you assigned to the parameter name field. The query string's value will be 'update_text', and along with it will be query strings who's name is the same as the ones you assigned in the Text field section of the Settings. These query strings will contain the updated text.
  • Replying to a message:
  • When a user replies to a message, your server-side script will recieve a query string who's name is the one you assigned to the parameter name field. The query string's value will be 'message_reply', and along with it will be query strings who's name is reply_message and email_address, and they will contain the reply message and email address of the initial message send, respectively.
  • Deleting a message:
  • When a user deletes a message, your server-side script will recieve a query string who's name is the one you assigned to the parameter name field. The query string's value will be 'delete_message', and along with it will be query strings who's name is 'message_id' which will contain the message id of the message to be deleted.
  • Deleting a single audio:
  • When a user deletes an audio, your server-side script will recieve a query string who's name is the one you assigned to the parameter name field. The query string's value will be 'delete_audio', and along with it will be query strings who's name is 'audio_path' which will contain the path of the audio to be deleted.
  • Deleting an image:
  • When a user deletes an image, your server-side script will recieve a query string who's name is the one you assigned to the parameter name field. The query string's value will be 'delete_image', and along with it will be query strings who's name is 'image_path' which will contain the path of the audio to be deleted.
  • Uploading an image:
  • When a user uploads an image, your server-side script will recieve a query string who's name is the one you assigned to the parameter name field. The query string's value will be 'upload_images', and if you are using PHP as you server-side script, the images will be in $_FILES['images'].
  • Deleting a video:
  • When a user deletes an video, your server-side script will recieve a query string who's name is the one you assigned to the parameter name field. The query string's value will be 'delete_video', and along with it will be query strings who's name is 'video_path' which will contain the path of the audio to be deleted.
